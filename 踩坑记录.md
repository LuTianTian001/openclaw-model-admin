# OpenClaw Model Admin / PocketClaw 踩坑记录

本文汇总在 **Model Admin 面板**、**OpenClaw 网关** 与 **PocketClaw（webchat）** 联调时真实遇到的问题、根因与处理方式，便于后续自己与协作者避坑。部署与权限类细则仍以 **[操作手册.md](./操作手册.md)** 为准。

---

## 1. 管理端网页「像连不上 API」、控制台报错后整页失效

**现象**：刷新或加载后页面异常、下拉无数据，浏览器控制台出现对 `undefined` 取属性的错误。

**根因**：`static/index.html` 里 `populate()` 使用 `groups.find(...).models`。当配置里的供应商名与前端分组键不一致（或找不到分组）时，`find` 返回 `undefined`，访问 `.models` 会直接抛错，后续 `renderFromState`、接口联动全部中断。

**处理**：

- 对 `find` 结果做兜底，例如：`(groups.find(...) || {}).models || []`，避免无匹配时崩溃。
- `apiReq` 对非 JSON 的 HTTP 响应应抛出明确错误，便于区分「接口挂了」与「数据为空」。
- 服务端 `GET /api/state` 对 `build_state()` 建议包一层 try/except，避免内部异常直接 500 导致前端无法解析。
- 避免在 `refresh()` 成功分支里无条件隐藏顶栏等 UI，以免盖住 `renderFromState` 本应展示的告警/错误条。

## 2. 自定义供应商 ID 大小写

**约定**：**自定义供应商标识一律使用小写**（配置、保存、迁移一致），避免同一逻辑供应商在 `openclaw.json`、`sessions.json`、路由 ref 里出现多种写法导致「找不到分组 / 模型不在列表」。

**处理**：写入配置前做迁移与归一（`models.providers`、`agents.defaults.models`、路由 primary/fallbacks、`agents` 子树、`auth.profiles`、各 `agents/*/agent/models.json` 等）；校验通过后对 `sessions.json` 中的 `providerOverride` 等字段做对应小写归一。具体实现以仓库内 `server.py` 当前逻辑为准。

## 3. PocketClaw 切模型：`GatewayRequestError: model not allowed: openai-codex/claude-opus-4-6`

**现象**：在 PocketClaw 里设置模型失败，网关返回 `model not allowed`，ref 形如 `openai-codex/claude-opus-4-6`。

**根因（两层）**：

1. **白名单**：OpenClaw 允许的模型集合来自配置里 **`agents.defaults.models` 的每一个 key**（完整 `provider/model`）。未出现在该对象中的 ref 会报 `model not allowed`。
2. **语义**：`openai-codex` 表示 Codex/OpenAI 线路；`claude-opus-4-6` 是 Claude 系模型 ID。二者拼在一起通常**既不在你的白名单里，也不对应真实上游**，属于错误组合。

**正确做法**：在承载 Claude 的供应商下选择模型，例如配置里若已声明 **`vx001/claude-opus-4-6`**，则客户端应使用 **`vx001`** + **`claude-opus-4-6`**，完整 ref 为 **`vx001/claude-opus-4-6`**。不要在 UI 里把「当前选中的供应商」与「另一个供应商下的模型 id」硬拼。

**不建议**：为消掉报错而把 `openai-codex/claude-opus-4-6` 强行写进 `agents.defaults.models`——即使过白名单，上游按 Codex 去调也会失败。

---

## 4. `models` 里的 `alias` 救不了「带斜杠的错 ref」

OpenClaw 对 **`agents.defaults.models` 条目的 `alias` 字段**：解析用户输入时，**仅当模型字符串不含 `/` 时**才会走别名表；已写成 `provider/model` 的形式会按字面解析，**不会**通过别名重定向到正确 provider。

因此错写成 `openai-codex/claude-opus-4-6` 时，不能指望给 `vx001/claude-opus-4-6` 配个别名就自动纠正。

## 5. `main·PocketClaw` 会不会跟着改「全局默认模型」变？

**机制**：

- **会话**在存储里对应键 **`agent:main:main`**（主 agent 的默认 webchat 直连会话）。
- **渠道**是 **`webchat`**；**PocketClaw** 一般出现在 `origin.label`，表示客户端名称，不是单独一种 channel 类型。

**是否跟 `agents.defaults.model.primary` 走**：

- 若该会话条目上**没有** `providerOverride` / `modelOverride`，则会话使用全局默认（随你改 primary 而变）。
- 若**有**这两项，则该会话**固定**使用这对 provider/model，**不随**全局 primary 变化。
- 在客户端里选择「恢复默认 / 使用全局默认」一类操作时，网关会**删除**这两项，会话重新跟全局走。

**仍可换模型**：会话覆盖不是锁死；在 PocketClaw 里再选其他**已在白名单内**的模型，会**改写** `providerOverride` / `modelOverride`。若客户端拼错 ref 或模型不允许，仍会报错。

## 6. 修改 `sessions.json` 的注意事项

- 与网关**并发写入**可能互相覆盖；大改时建议**短暂停止网关**或确认无活跃会话写入后再改。
- 修改后执行 **`systemctl restart openclaw-gateway`**（或你的等价重启方式）可确保行为与内存状态一致（视版本而定，重启更稳妥）。
