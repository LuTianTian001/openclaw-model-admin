# OpenClaw Model Admin 部署与排障操作手册

面向**需要把面板跑起来、并理解遇到问题怎么办**的读者。建议按章节顺序第一次部署；排障可直接跳转到 [§12 常见问题](#12-常见问题与解决办法) 按现象查找。

**平台：仅 GNU/Linux**（`uname -s` 为 `Linux`）。面板与安装脚本不会在 macOS / Windows 上运行。

## 2. 核心概念（读一遍可少踩坑）

- **数据目录**：OpenClaw 的 `openclaw.json`、会话等所在目录。默认是**当前运行用户的** `~/.openclaw/`。
- **用户一致性**：运行 OpenClaw 网关的 **Linux 用户**与运行本面板的用户**必须是同一用户**（或显式把 `OPENCLAW_HOME` / `OPENCLAW_CONFIG_PATH` 指到**网关正在使用的那份文件**，且进程对文件有读写权）。否则你改的是「另一套主目录下的配置」，网关不会变。
- **会话路径**：一般**不要手写**。程序默认使用：  
  `<openclaw.json 所在目录>/agents/main/sessions/sessions.json`  
  只有目录结构特殊时才设 `OPENCLAW_SESSIONS_PATH`。
- **校验**：保存配置前会执行 `openclaw config validate`。没有 CLI 时（常见于 Docker 镜像）需 `OPENCLAW_MODEL_ADMIN_SKIP_VALIDATE=1`。
- **重启网关**：默认执行 `systemctl restart <OPENCLAW_GATEWAY_SERVICE>`。没有 systemd 或权限不足时，必须配置 `OPENCLAW_GATEWAY_RESTART_COMMAND`。

## 3. 部署前检查清单

在目标机器上依次执行（可复制整段）。

```bash
# 3.1 必须是 Linux
uname -s    # 期望输出 Linux

# 3.2 Python 版本 >= 3.10
python3 -c 'import sys; print(sys.version); assert sys.version_info >= (3,10)'

# 3.3 是否存在默认配置（若不存在，先完成 OpenClaw 初始化，或准备好自定义路径）
ls -l ~/.openclaw/openclaw.json

# 3.4 是否与网关同一用户（示例：网关以用户 openclaw 运行，则你也应用该用户装面板）
whoami
id

# 3.5 openclaw CLI（宿主机直跑面板时强烈建议有）
command -v openclaw && openclaw --version || echo "无 openclaw：Docker 或需 SKIP_VALIDATE"

# 3.6 一键脚本依赖（默认 git 路径）
command -v git || echo "无 git：使用 USE_GIT=0"
```

## 4. 一键安装与启动

### 4.1 默认方式（有 git）

```bash
curl -fsSL https://raw.githubusercontent.com/LuTianTian001/openclaw-model-admin/main/install.sh | bash
```

默认安装目录：`$HOME/openclaw-model-admin`。

### 4.2 无 git（仅 curl/wget + tar）

```bash
curl -fsSL https://raw.githubusercontent.com/LuTianTian001/openclaw-model-admin/main/install.sh | USE_GIT=0 bash
```

### 4.3 常用环境变量（与一键命令组合）

管道执行的子 shell **不会继承**你当前终端里 `export` 的变量，除非写在命令前面：

```bash
curl -fsSL https://raw.githubusercontent.com/LuTianTian001/openclaw-model-admin/main/install.sh \
  | INSTALL_DIR=/opt/openclaw-model-admin OPENCLAW_HOME=/var/lib/openclaw bash
```

| 变量 | 含义 |
|------|------|
| `INSTALL_DIR` | 安装目录，默认 `$HOME/openclaw-model-admin` |
| `REPO` | GitHub `owner/repo`，默认 `LuTianTian001/openclaw-model-admin` |
| `BRANCH` | 分支，默认 `main` |
| `USE_GIT` | `1` 用 git，`0` 用源码包 |
| `OPENCLAW_HOME` / `OPENCLAW_CONFIG_PATH` | 与面板读取的配置一致（见 §5） |
| `SKIP_OPENCLAW_CHECK` | `1` 时不提示缺少 `openclaw` |
| `INSTALL_COPY_ENV` | `0` 禁止自动复制 `.env.example` → `.env`（默认 `1` 会在无 `.env` 时生成模板） |

**一键脚本会自动做：**

- 拒绝非 Linux。
- 检查 Python ≥ 3.10。
- 提示默认路径下是否存在 `openclaw.json`、是否存在 `openclaw` 命令。
- 克隆或同步代码、`chmod +x`、启动 `start.sh`。
- 若安装目录无 `.env` 且存在 `.env.example`，**复制为 `.env`** 方便你按注释填写（可通过 `INSTALL_COPY_ENV=0` 关闭）。

**一键脚本不会替你做的：** 见 [§11](#11-一键无法自动完成的事项须手动处理)。

## 5. 配置路径：OPENCLAW_HOME 与 OPENCLAW_CONFIG_PATH

优先级（与程序一致）：

1. 若设置了 **`OPENCLAW_CONFIG_PATH`**（绝对路径），用它作为 `openclaw.json`。
2. 否则若设置了 **`OPENCLAW_HOME`**，使用 `$OPENCLAW_HOME/openclaw.json`。
3. 否则使用 **`$HOME/.openclaw/openclaw.json`**。

**推荐**：数据在自定义目录时只设 **`OPENCLAW_HOME=/实际路径`**，不必再写 `OPENCLAW_SESSIONS_PATH`（除非目录结构不是 `.../agents/main/sessions/sessions.json`）。

在面板安装目录创建或编辑 **`.env`**（`start.sh` 自动 `source`）：

```bash
cd ~/openclaw-model-admin   # 或你的 INSTALL_DIR
cp -n .env.example .env     # 若一键已复制可跳过
nano .env
```

修改后重新执行 `./start.sh`（或重启 systemd 服务）。

## 6. 首次启动后验证

1. 浏览器打开：`http://127.0.0.1:8765`（若 `OPENCLAW_MODEL_ADMIN_HOST=0.0.0.0`，可用本机局域网 IP + 端口）。
2. 页面应能加载；若顶部有红色告警，先读告警文案。
3. 本机可测 API（可选）：

```bash
curl -sS http://127.0.0.1:8765/api/state | head -c 400
echo
```

若返回 JSON 且含 `configPath`、`gatewayActive` 等字段，说明 HTTP 服务正常。

## 8. systemd 常驻（详细步骤）

目标：开机自启、崩溃重启，且 **User= 与网关相同**。

1. 复制示例单元：

```bash
sudo cp openclaw-model-admin.service.example /etc/systemd/system/openclaw-model-admin.service
```

2. 编辑该文件，至少修改：
   - `User=`：与运行 OpenClaw 网关的用户一致。
   - `WorkingDirectory=`：面板代码目录（内含 `server.py`）。
   - `ExecStart=`：建议改为该目录下 `python3 server.py` 的**绝对路径**，或与 `WorkingDirectory` 一致使用 `/usr/bin/python3 server.py`（以示例文件为准）。
   - `Environment=OPENCLAW_HOME=...`：指向该用户的 OpenClaw 数据根目录。

3. 加载并启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now openclaw-model-admin
sudo systemctl status openclaw-model-admin
```

4. 看日志：

```bash
journalctl -u openclaw-model-admin -f
```

若 `status` 为 failed，常见原因是 **User 无权限读 `OPENCLAW_HOME`** 或 **Python 路径不对**。

## 9. Docker / Compose（Linux 宿主机）

### 9.1 基本步骤

```bash
cp docker-compose.example.yml docker-compose.yml
```

编辑 `docker-compose.yml`：

- **`volumes`**：把宿主机上的 **整个** OpenClaw 数据目录挂载到容器内路径（与 `OPENCLAW_CONFIG_PATH` 一致）。不要只挂单个 `openclaw.json` 文件，否则相对路径下的 `agents/` 可能不在预期位置。
- **`OPENCLAW_MODEL_ADMIN_SKIP_VALIDATE`**：镜像内通常没有 `openclaw`，示例中设为 `1` 是常见做法。

```bash
docker compose up --build
```

### 9.2 网关跑在宿主机、面板跑在容器（最常见难点）

- 容器里的 **`127.0.0.1` 是容器自己**，不是宿主机。  
  **解决办法**：把 `OPENCLAW_GATEWAY_HEALTH_URL` 设为宿主机在**容器网络内可达**的地址，例如：
  - Docker Desktop / 部分环境：`http://host.docker.internal:端口/`
  - Linux 可尝试网关监听地址为 `0.0.0.0` 时，用宿主机局域网 IP：`http://192.168.x.x:18789/`（端口以你实际为准）
- **面板内「重启网关」**：容器内默认没有宿主机的 systemd。  
  **解决办法**：设置 `OPENCLAW_GATEWAY_RESTART_COMMAND` 为一条在**该容器内有权限执行**且能重启宿主机进程/容器的命令（例如挂载 `docker.sock` 后执行 `docker restart 网关容器名`；或通过 `ssh` 到宿主机执行 `systemctl`——具体以你的安全策略为准）。若不配置，请在宿主机上手动重启网关。

### 9.3 网关与面板在同一 Docker 主机上、两个服务都是容器

- 用 **Docker 网络**或 **compose 服务名** 互访，把 `OPENCLAW_GATEWAY_HEALTH_URL` 写成 `http://网关服务名:端口/`。
- 重启命令可以是 `docker restart ...`（需权限）或 `docker compose restart 服务名`。

## 10. 环境变量完整说明

与 `README.md`、`.env.example` 一致，此处强调**何时必须改**：

| 变量 | 何时必须改 |
|------|------------|
| `OPENCLAW_HOME` | 数据不在 `~/.openclaw` |
| `OPENCLAW_CONFIG_PATH` | 配置文件路径与 `OPENCLAW_HOME/openclaw.json` 规则不一致 |
| `OPENCLAW_SESSIONS_PATH` | 会话文件不在默认推导路径下 |
| `OPENCLAW_GATEWAY_SERVICE` | systemd 单元名不是 `openclaw-gateway.service` |
| `OPENCLAW_GATEWAY_RESTART_COMMAND` | 无 systemd、或 `systemctl` 无权限、或 Docker 内重启 |
| `OPENCLAW_GATEWAY_HEALTH_URL` | Docker/无 systemd 时需要可靠「在线」判断；自定义重启后要做就绪探测 |
| `OPENCLAW_GATEWAY_SS_MARKERS` | 网关监听端口不是默认、且未配 HEALTH_URL |
| `OPENCLAW_MODEL_ADMIN_SKIP_VALIDATE` | 环境里没有 `openclaw` 可执行文件 |
| `OPENCLAW_MODEL_ADMIN_HOST` / `PORT` | 端口冲突或仅本机访问时改为 `127.0.0.1` |

## 11. 一键无法自动完成的事项（须手动处理）

下列事项依赖你的系统架构与安全策略，脚本**无法**安全地替所有人自动决定，请按说明操作：

| 事项 | 原因 | 推荐做法 |
|------|------|----------|
| 选对 **Linux 用户** | 多用户系统上主目录隔离 | 用与网关相同用户安装；systemd 里设置相同 `User=` |
| **防火墙** 放行面板端口 | 脚本不应擅自改 iptables/nftables | 自行放行 `OPENCLAW_MODEL_ADMIN_PORT`；生产建议仅内网或反代 + TLS |
| **TLS / HTTPS** | 需要证书与域名 | 用 Nginx/Caddy 等反向代理到 `127.0.0.1:8765` |
| **Docker 内重启宿主机网关** | 涉及 socket、ssh、权限 | 自行设计 `OPENCLAW_GATEWAY_RESTART_COMMAND` 并限制权限 |
| **网关 HTTP 端口/路径** | 各版本可能不同 | 用 `curl`/`ss` 确认监听地址后填写 `OPENCLAW_GATEWAY_HEALTH_URL` 或 `OPENCLAW_GATEWAY_SS_MARKERS` |
| **备份** | 数据归属用户 | 定期备份 `openclaw.json` 与同目录下 `agents/` |
| **curl \| bash 的信任** | 供应链风险 | 可改为先下载 `install.sh` 审阅后再 `bash install.sh` |

## 12. 常见问题与解决办法

### Q1：执行 install 提示「仅支持 Linux」

- **原因**：当前内核不是 Linux。  
- **处理**：在 Linux 虚拟机/服务器上部署；本仓库不按 macOS/Windows 适配。

### Q2：Python 版本过低

- **现象**：`install.sh` 报 Python 不足 3.10。  
- **处理**：安装 Python 3.10+（发行版包管理器或 pyenv）；保证 `python3` 指向新版本。

### Q3：页面能开，但保存配置失败，提示校验错误

- **现象**：与 `openclaw config validate` 相关。  
- **处理**：在**同一用户** shell 下运行 `openclaw config validate` 看具体错误；修正 `openclaw.json`。若暂时无法装 CLI（如容器内），设 `OPENCLAW_MODEL_ADMIN_SKIP_VALIDATE=1`（**接受跳过校验的风险**）。

### Q4：提示找不到 `openclaw.json` 或改了不生效

- **原因**：面板读的路径与网关不是同一份文件。  
- **处理**：在面板页头或 `/api/state` 的 `configPath` 核对路径；设置正确的 `OPENCLAW_HOME` 或 `OPENCLAW_CONFIG_PATH`；确认用户一致。

### Q5：「重启网关」失败

- **原因**：无 systemd、服务名不对、当前用户无权 `systemctl`。  
- **处理**：  
  - 核对 `OPENCLAW_GATEWAY_SERVICE`；  
  - 或配置 `OPENCLAW_GATEWAY_RESTART_COMMAND`（如 `sudo systemctl restart openclaw-gateway.service`，并配置 sudoers）；  
  - 或在宿主机手动重启网关。

### Q6：页面显示网关离线，但网关实际在跑

- **原因**：`systemctl` 不可用且 `ss` 匹配不到端口；或 Docker 内误判。  
- **处理**：设置 **`OPENCLAW_GATEWAY_HEALTH_URL`** 为可访问的网关 URL；或设置 **`OPENCLAW_GATEWAY_SS_MARKERS`** 包含实际监听地址片段。

### Q7：端口 8765 已被占用

- **现象**：启动失败或冲突。  
- **处理**：在 `.env` 设置 `OPENCLAW_MODEL_ADMIN_PORT=其他端口`，重启面板。

### Q8：`curl | bash` 时环境变量没生效

- **原因**：变量必须写在管道左侧传给 bash。  
- **处理**：  
  `curl ... | OPENCLAW_HOME=/path bash`  
  或安装完成后在 **`安装目录/.env`** 里写变量，再 `./start.sh`。

### Q9：git pull 失败（本地改过文件）

- **现象**：`install.sh` 更新阶段失败。  
- **处理**：在 `INSTALL_DIR` 内 `git status` 处理冲突；或临时 `USE_GIT=0` 用源码包同步（注意保留 `.env`）。

### Q10：无 git、无 rsync

- **现象**：`USE_GIT=0` 时提示无 rsync 使用 cp。  
- **处理**：一般仍可完成；若异常可安装 `rsync` 后重试。

## 13. 备份与升级

**备份（操作前建议）：**

```bash
cp -a ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.bak.$(date +%Y%m%d)
cp -a ~/.openclaw/agents ~/.openclaw/agents.bak.$(date +%Y%m%d)   # 若需会话级回滚
```

**升级面板代码：**

- 使用 `install.sh` 且 `USE_GIT=1`：再次执行同一条 `curl | bash` 会 pull 并启动。  
- 手动克隆：在仓库目录 `git pull` 后 `./start.sh`。

## 14. 安全提示

- 勿在公开渠道粘贴 **GitHub PAT**、API Key；泄露须立即撤销。  
- 面板默认监听 `0.0.0.0` 时，局域网内可达，请结合防火墙与反代控制暴露面。  
- `OPENCLAW_GATEWAY_RESTART_COMMAND` 具备 shell 执行能力，仅写入可信内容。
